# Swift Code Style Guide

- Follow Apple's official Swift API Design Guidelines.
- Prefer Structs over Classes for models and state management.
- Use SwiftUI for all UI components.
- Adhere to the MVVM architectural pattern.
- Prioritize Swift Structured Concurrency (Async/Await).
- Use Guard statements for early exits and clear unwrapping.
- Document all public APIs with clear DocC comments.
